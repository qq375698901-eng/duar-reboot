## 当前设计基调
- 战斗是高风险、低回复的动作 PVE / PVP 兼容框架。
- 玩家在战斗流程中应尽量避免 HP 损失，而不是依赖频繁回复。
- 敌人设计优先制造“读招、走位、停手、换目标”的决策压力，不走无双割草路线。
- 地牢战斗强调谨慎推进、短时高压和明确惩罚。

## 小怪策划口径

### 设计原则
- 每只小怪承担单一、明确、可读的战术压力。
- 单怪不追求复杂机制，组合时才产生深度。
- 优先做“逼玩家改变行为”的怪，而不是单纯堆数值的怪。

### 小怪阵容与定位
- 地牢刀手：基础正面近战杂兵，稳定逼近，单段挥砍，负责建立基础近战节奏。
- 突刺兵：单段快速前刺，负责抓玩家空档、惩罚贪刀和站桩。
- 人形弩手：后排逼位单位，负责迫使玩家切目标与移动。
- 盾卫：正面防反单位，核心是“正面打会被立刻反咬，绕后有窗口”。
- 地牢猎犬：低身位扑咬野兽，负责贴地机动与脚下压力。
- 火瓶投掷手：后排控场单位，负责地面封锁与路线压缩。

### 当前开发顺序结论
1. 地牢刀手
2. 突刺兵
3. 人形弩手
4. 盾卫
5. 地牢猎犬
6. 火瓶投掷手

## 精英敌人口径

### 第一只精英怪
- 名称：狱庭监工
- 定位：房间核心威胁
- 气质：慢、重、范围大、压迫强
- 武器：双手长柄处刑斧
- 动画：逐帧贴图，不继承玩家分件动画

### 战斗逻辑
- 缓慢推进，不高速贴脸
- 主要靠大范围横扫、重下劈、慢回身挥击制造压迫
- 必须给明显前摇和恢复窗口
- 不做法术、召唤、冲锋、多阶段狂暴

### 推荐精英房组合
- 狱庭监工 + 地牢刀手 x2
- 狱庭监工 + 人形弩手 + 地牢刀手
- 狱庭监工 + 盾卫 + 地牢猎犬

## Boss 口径

### 第一只 Boss
- 名称：石柱五面像
- 技术结构：5 个独立头部单位 + 1 个 Boss Controller
- 石柱主体不属于怪物交付内容，已经做到 `boss_room_1_image_map` 背景里
- 只有全部头部被击破，Boss 才算死亡

### 五个头的功能
- 风头：
  - 施加减速
  - 不吹飞
  - 不拖拽
- 火头：
  - 看向左 / 右平台
  - 用眼睛颜色提示本轮绑定头
  - 只有绑定头成功发动技能后才开始 3 秒倒计时
  - 倒计时结束后，对应平台爆炸
  - 若绑定头等待期间死亡，本轮作废
- 雷头：
  - 生成 10 个落雷预警
  - 1.5 秒后结算
  - 只伤害落点上方整条垂直线
  - 落点以下无伤害
- 守卫头：
  - 给其他一个头施加反伤
  - 玩家打到被标记头时，自己也会受伤
- 恢复头：
  - 对其他一个头持续治疗
  - 自身受到玩家伤害时，治疗立刻打断

### 火头绑定颜色
- 白色：风头
- 绿色：恢复头
- 黑色：守卫头
- 黄色：雷头

### 当前 Boss 正式行为逻辑
- Boss 在玩家未攻击任意头之前统一保持 `IDLE`
- 玩家第一次攻击任意一个头后，Boss 正式开战
- 开战后启动一个全局战斗计时器
- 全局计时器决定同一时间允许激活几个头，以及是否存在公共施法冷却

### Boss 战斗阶段
- 第一阶段 `0:00 - 1:00`
  - 同时只激活 1 个头
  - 每轮技能结束后进入 3 秒公共冷却
- 第二阶段 `1:00 - 2:00`
  - 同时只激活 2 个头
  - 每轮技能结束后进入 3 秒公共冷却
  - 火头不能连续激活，必须等自身完整流程结束后才重新入池
- 第三阶段 `2:00 - 3:00`
  - 同时只激活 3 个头
  - 每轮技能结束后进入 3 秒公共冷却
  - 火头规则同第二阶段
- 第四阶段 `3:00+`
  - 同时只激活 3 个头
  - 取消公共施法冷却
  - 火头规则同第二阶段

### 技能流程结束判定
- 风头：本轮风场结束
- 恢复头：治疗结束或被打断
- 守卫头：本轮反伤施加完成
- 雷头：预警与落雷结算完成
- 火头：整轮“看向 + 绑定 + 等待触发 + 爆炸 / 作废”结束

### Boss 实现原则
- 头部不再各自自由循环施法
- 所有激活都由 `judgement_boss_controller` 调度
- 已死亡头不能再次进入随机池
- 头死亡不会重置全局战斗计时器

- 浮窗使用独立 `CanvasLayer`，避免受父级 UI 缩放影响鼠标跟随位置

## 装备强化定义
- 强化等级当前定义为“强化次数”，范围为 `0 ~ 4`
- 每次使用强化石时，武器强化次数 `+1`
- 强化带来的具体属性不再写死为统一百分比，而是由本次使用的强化石决定
- 当前已为装备数据正式预留强化加成字段：
- 基础攻击力加成 `base_attack_power`
- 基础防御力加成 `base_defense_ratio`
- 额外人物属性加成 `attack / agility / vitality / spirit`
- 当前实现中，强化加成按聚合结果保存在 `reinforcement_bonus`，并额外预留 `reinforcement_history` 以便后续记录每一次强化石来源
- 当前实现假设：你提到的“力量 / 敏捷 / 体魄 / 意志”先映射到现有人物四维口径 `attack / agility / vitality / spirit`
- 装备的有效基础攻击力、有效基础防御力现在都会计入强化加成；背包、主城人物面板和战斗中武器实例都会读取强化后的有效值

## 第一批强化石
- 已接入两种一阶强化石：
- `T1 Reinforce Stone (ATK)`：使武器基础攻击力随机增加 `1 ~ 3`
- `T1 Reinforce Stone (DEF)`：使武器基础防御力随机增加 `1% ~ 3%`
- 追加四种一阶属性强化石：
- `T1 Reinforce Stone (Power)`：额外力量 `+1 ~ 2`
- `T1 Reinforce Stone (Agility)`：额外敏捷 `+1 ~ 2`
- `T1 Reinforce Stone (Vitality)`：额外体魄 `+1 ~ 2`
- `T1 Reinforce Stone (Spirit)`：额外意志 `+1 ~ 2`
- 当前强化石作为正式物品类型 `reinforcement_stone` 接入库存系统，并拥有独立图标
- 当前测试库存中已默认放入一批攻击/防御强化石，便于直接验证流程
- 当前实现中，力量 / 敏捷 / 体魄 / 意志 对应程序字段 `attack / agility / vitality / spirit`
- 当前交互口径：
- 在主城库存中双击强化石，会打开强化小窗口
- 在战局背包中双击强化石，也会打开强化小窗口
- 强化窗口中可选择目标武器，点击确认后会消耗该强化石、落实随机强化值，并使目标武器强化次数 `+1`
- 武器达到 `4/4` 后，不再允许作为强化目标
- 装备悬停浮窗当前已显示强化次数，格式为 `0/4 ~ 4/4`
- 若装备存在四维属性加成，装备浮窗也会额外显示 `Power / Agility / Vitality / Spirit` 对应的加值
- 主城人物界面中的四维属性当前按“基础值 + 装备加值”显示；例如装备提供力量 `+3` 时，人物界面会显示为 `基础值 +3`

## 药水系统
- 新增正式药水物品类型 `potion`，并新增独立装备槽 `equipped_potion`
- 当前药水槽与武器槽分离保存，且同样纳入账号库存存档
- 药水当前支持堆叠，默认最大堆叠数为 `10`
- 背包新增堆叠合并逻辑：同种药水在背包/仓库移动、调试添加、装备/卸下时都会优先自动叠堆
- 新增首个药水道具 `T1 Red Potion`
- 图标路径：`art/items/potion_t1_red.png`
- 当前效果：恢复 `30 HP`
- 当前使用前摇：`3.0s`
- 当前使用代价口径：一旦开始使用，药水数量立刻 `-1`；若后续被受击打断，不返还药水
- 当前使用状态口径：
- 药水使用被视为一种人物操作状态
- 处于药水使用状态时，禁止任何装备操作 / 武器操作
- 药水使用中人物仍可移动和跳跃，但移动速度与跳跃力均降低 `70%`（当前实现为保留 `30%`）
- 药水使用会在人物头顶显示所用药水的图标
- 进入受击相关状态时，药水使用会被打断；当前已覆盖僵直、击飞、抓取这三类中断入口
- 战局内新增快捷键 `G`：使用当前药水槽中携带的药水
- 主城库存与战局背包界面均已补充药水装备槽
- 当前交互口径：
- 在背包中选中药水，再点击药水槽，可将药水带到药水装备栏
- 点击药水装备槽可卸下药水回背包
- 战局中若人物正在喝药，局内背包中的穿戴/移动操作也会被锁住，保持“不能进行任何装备操作”的统一口径
- 装备悬停浮窗已补充药水信息显示：当前至少显示堆叠数、恢复量、使用时间

## 人物死亡状态
- 玩家当前已补齐正式死亡状态 `DEAD`
- 当前死亡触发口径：`HP <= 0` 时立即进入死亡状态
- 死亡时会立刻中断所有装备操作、技能动作、格挡、喝药等人物操作状态
- 死亡时会清空当前武器相关动作残留，并隐藏已装备武器显示
- 玩家当前新增独立 `death` 动画，区别于普通倒地 `down`
- 死亡时会覆盖到正式死亡姿态，不再继续播放原武器/技能动作
- 战局中会生成独立死亡遮罩层：界面会缓慢泛红，随后出现“回到城镇”按钮
- 点击“回到城镇”会结束当前地牢流程并返回主城界面
- 当前修正口径：
- 玩家处于 `DOWN` 或 `DEAD` 时，不再接受新的僵直、击飞、抓取或伤害受击链
- 怪物处于 `DOWN` / `DEAD` / `GRABBED` 时，其攻击判定盒不会再继续造成伤害
- 当前死亡惩罚口径：
- 玩家进入死亡状态时，立即丢失背包内全部物品
- 玩家进入死亡状态时，立即丢失当前装备的武器与药水
- 当前仓库物品不会因死亡而丢失
- 战局内当前新增统一坠落即死线
- 当前由 `DungeonRoom2D` 向玩家下发坠落死亡阈值，默认值先设为 `y = 960`
- 玩家在战局地图中下落到该高度及以下时，会直接进入死亡状态

## 当前设计基调
- 战斗是高风险、低回复的动作 PVE / PVP 兼容框架。
- 玩家在战斗流程中应尽量避免 HP 损失，而不是依赖频繁回复。
- 敌人设计优先制造“读招、走位、停手、换目标”的决策压力，不走无双割草路线。
- 地牢战斗强调谨慎推进、短时高压和明确惩罚。

## 小怪策划口径

### 设计原则
- 每只小怪承担单一、明确、可读的战术压力。
- 单怪不追求复杂机制，组合时才产生深度。
- 优先做“逼玩家改变行为”的怪，而不是单纯堆数值的怪。

### 小怪阵容与定位
- 地牢刀手：基础正面近战杂兵，稳定逼近，单段挥砍，负责建立基础近战节奏。
- 突刺兵：单段快速前刺，负责抓玩家空档、惩罚贪刀和站桩。
- 人形弩手：后排逼位单位，负责迫使玩家切目标与移动。
- 盾卫：正面防反单位，核心是“正面打会被立刻反咬，绕后有窗口”。
- 地牢猎犬：低身位扑咬野兽，负责贴地机动与脚下压力。
- 火瓶投掷手：后排控场单位，负责地面封锁与路线压缩。

### 当前开发顺序结论
1. 地牢刀手
2. 突刺兵
3. 人形弩手
4. 盾卫
5. 地牢猎犬
6. 火瓶投掷手

## 精英敌人口径

### 第一只精英怪
- 名称：狱庭监工
- 定位：房间核心威胁
- 气质：慢、重、范围大、压迫强
- 武器：双手长柄处刑斧
- 动画：逐帧贴图，不继承玩家分件动画

### 战斗逻辑
- 缓慢推进，不高速贴脸
- 主要靠大范围横扫、重下劈、慢回身挥击制造压迫
- 必须给明显前摇和恢复窗口
- 不做法术、召唤、冲锋、多阶段狂暴

### 推荐精英房组合
- 狱庭监工 + 地牢刀手 x2
- 狱庭监工 + 人形弩手 + 地牢刀手
- 狱庭监工 + 盾卫 + 地牢猎犬

## Boss 口径

### 第一只 Boss
- 名称：石柱五面像
- 技术结构：5 个独立头部单位 + 1 个 Boss Controller
- 石柱主体不属于怪物交付内容，已经做到 `boss_room_1_image_map` 背景里
- 只有全部头部被击破，Boss 才算死亡

### 五个头的功能
- 风头：
  - 施加减速
  - 不吹飞
  - 不拖拽
- 火头：
  - 看向左 / 右平台
  - 用眼睛颜色提示本轮绑定头
  - 只有绑定头成功发动技能后才开始 3 秒倒计时
  - 倒计时结束后，对应平台爆炸
  - 若绑定头等待期间死亡，本轮作废
- 雷头：
  - 生成 10 个落雷预警
  - 1.5 秒后结算
  - 只伤害落点上方整条垂直线
  - 落点以下无伤害
- 守卫头：
  - 给其他一个头施加反伤
  - 玩家打到被标记头时，自己也会受伤
- 恢复头：
  - 对其他一个头持续治疗
  - 自身受到玩家伤害时，治疗立刻打断

### 火头绑定颜色
- 白色：风头
- 绿色：恢复头
- 黑色：守卫头
- 黄色：雷头

### 当前 Boss 正式行为逻辑
- Boss 在玩家未攻击任意头之前统一保持 `IDLE`
- 玩家第一次攻击任意一个头后，Boss 正式开战
- 开战后启动一个全局战斗计时器
- 全局计时器决定同一时间允许激活几个头，以及是否存在公共施法冷却

### Boss 战斗阶段
- 第一阶段 `0:00 - 1:00`
  - 同时只激活 1 个头
  - 每轮技能结束后进入 3 秒公共冷却
- 第二阶段 `1:00 - 2:00`
  - 同时只激活 2 个头
  - 每轮技能结束后进入 3 秒公共冷却
  - 火头不能连续激活，必须等自身完整流程结束后才重新入池
- 第三阶段 `2:00 - 3:00`
  - 同时只激活 3 个头
  - 每轮技能结束后进入 3 秒公共冷却
  - 火头规则同第二阶段
- 第四阶段 `3:00+`
  - 同时只激活 3 个头
  - 取消公共施法冷却
  - 火头规则同第二阶段

### 技能流程结束判定
- 风头：本轮风场结束
- 恢复头：治疗结束或被打断
- 守卫头：本轮反伤施加完成
- 雷头：预警与落雷结算完成
- 火头：整轮“看向 + 绑定 + 等待触发 + 爆炸 / 作废”结束

### Boss 实现原则
- 头部不再各自自由循环施法
- 所有激活都由 `judgement_boss_controller` 调度
- 已死亡头不能再次进入随机池
- 头死亡不会重置全局战斗计时器

## 美术与动画口径
